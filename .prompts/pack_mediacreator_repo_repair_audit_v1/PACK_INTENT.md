# Pack intent

Repair the repo to the point where the app behaves honestly and deterministically for the first real operator journey:

1. upload labeled photos,
2. see truthful progress,
3. get correct photo routing,
4. create a saved character,
5. create a saved 3D output with truthful stage labels,
6. train and use a LoRA only when the runtime is truly capable,
7. save a proof image only when it actually exists.

This pack must prefer truth over optimism.
If a capability is not implemented or not operational, the code and UI must say so plainly.
